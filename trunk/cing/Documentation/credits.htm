The Python software package CING, pronounced “king”, is an acronym for Common Interface to NMR structure Generation. CING integrates its analyses with many external biomolecular tools: WHAT_CHECK (1), PROCHECK/Aqua (2), XPLOR-NIH (3), Wattos (4), CYANA (5), DSSP (6), MOLMOL (7), ShiftX (8) and transparently interfaces to the CCPN (9) framework. It also builds on projects such as: MatPlotLib, Povray, and Cython. CING can read data from over 4,000 NMR-solved protein and nucleic-acid structures deposited at PDB and annotated and processed in the NMR Restraints Grid (4) at the BMRB (10).

 

The development of reliable, smart, well-documented and foremost useful computational tools for structure calculation and structure validation has become increasingly important for NMR-based structural projects (11; 12). Unfortunately, at present no single program can tell you if your results are correct. Existing tools such as QUEEN (13) can however assist in finding unlikely elements in the data and structure ensembles and other tools demonstrated by the DRESS (14) and RECOORD (15) projects can improve the results.

 

We will present the integrated software package that is based upon our philosophy of a residue-based structure validation (11). CING generates per-residue validation summaries in HTML formatted pages that can be inspected using any common web browser. CING will eventually also integrate the QUEEN version 2, including the residue-specific analysis (13; 16).
